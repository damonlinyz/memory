# 人类记忆系统架构设计讨论

## 讨论日期
2026-05-29

## 核心主题
什么样的记忆系统能真正模拟人类记忆的特性：
- 关联网络
- 上下文相关
- 模糊检索
- 情感权重
- 动态演化
- 多模态

## 关键洞察

### 1. 记忆 = 主动构建 + 被动积累

- **主动知识**：用户主要活动/学习中主动获取和记录的
- **被动卷入**：环境碎片（走路时看到的行人、衣服颜色、红绿灯等）

例：过马路等红绿灯时，被动看到的信息（行人颜色、帽子）会在某天被某个场景触发召唤出来。

### 2. 关联是构建出来的

- 构建过程是主动的
- 环境记忆会被连带卷入
- 平时是浅记忆，触发时临时主动构建出来

### 3. 知识在物理世界是可视化的

- 人类记忆形成突触组织
- 记忆系统可能需要某种空间/拓扑性质
- 关联不是查表，而是"距离或方向"的关系

### 4. 触发机制 - 行动流触发

触发不是"搜索"，而是**人在做某件事的过程中**自然发生的。

触发条件 = **关键词 + 场景 + 活动** 三者共同构成

- 当前活动：设计师在做服装设计
- 当前场景：工作坊/对话
- 关键词：红色、衣服

→ 触发历史记忆碎片：某天过马路时看到的红色衣服

关键洞察：记忆不是被"查询"唤醒的，而是被**当前行动流**自动卷进来的。

因此：
- 记录时需要绑定**活动上下文**（不只是内容）
- 检索时基于**当前活动**而非关键词匹配

2. **构建过程** → 已解决：行动导向的按需卷入

- 围绕当前行动，按需提取和注入
- 不是随机扩散，而是**按需召唤**
- 碎片平时积累，需要时临时组装
- 设计师需要红色知识 → 召唤"过马路看到的红衣服"
3. **记忆的"重量"** → 已解决：艾宾浩斯遗忘曲线机制

权重 = f(使用频率, 距今时间)

- **强化**：重复越多，权重越高
- **积累**：短频快 → 转化为长期记忆（如单词背诵）
- **衰减**：长时间不用，按艾宾浩斯曲线遗忘

系统需要：
- 使用计数（每次调用+1）
- 衰减函数（时间指数衰减）
- 转化阈值（短期用多→长期）

## 生物学机制映射

### 1. 睡眠巩固

- **触发**：用户长时间无操作 → 判定"睡眠" → 凌晨批处理
- **初始策略**：根据用户时区预设睡眠窗口
- **演进**：观察用户行为习惯，逐步调整窗口
- **执行**：
  - 碎片 → 结构化记忆
  - 发现碎片间的新关联（类梦机制）
  - 遗忘不必要细节
  - 核心动作：强化权重 + 知识归档

### 2. 情绪印记

**隐式情绪信号（无需用户主动输入）：**

| 信号来源 | 提取方式 |
|---------|---------|
| 语气词 | 文本情感分析 |
| 语音语气 | 声调、语速、停顿 |
| 打字节奏 | 速度变化、删除频率 |
| 行为反馈 | 点赞、点踩、纠偏 |

**情绪记录结构：**

```
emotion_tag: {
  baseline: "今天整体情绪"（小时级采样），
  events: [
    { time: "14:32", peak: true, valence: -0.7, arousal: 0.9 }
  ]
}
```

记忆绑定到**情绪事件**（峰值），而非情绪基线。

### 3. 空间位置绑定

**核心观点：数字空间位置 > 物理空间位置**

物理空间依赖用户授权，非核心。
数字空间是用户行为的实际载体。

```
记忆锚点 = {
  session_id: "sess_20260529_设计讨论",
  digital_context: {
    app: "Claude Code",
    channel: "CLI对话",
    topic: "服装设计"
  }
}
```

Session作为记忆单元：天然聚合完整行动流，跨session检索时可过滤。
